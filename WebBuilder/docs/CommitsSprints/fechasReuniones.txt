Nombre de la aplicación: WebBuilder.
Funcionalidad principal:
WebBuilder es una aplicación web cuyo objetivo es procesar los datos 
proporcionados por una API externa para generar automáticamente una página 
web personalizada, basada en las preferencias del usuario.
Fases de desarrollo del proyecto:
1. Creación de la página principal (landing page)
Esta página inicial explicará la funcionalidad de la aplicación e incluirá opciones 
para que el usuario pueda registrarse o iniciar sesión. Tendrá algún botón para 
comenzar que llevará a la página donde se introducirá el enlace.
Desde esta página se podrá introducir el enlace de la API y, si la validación es 
correcta se podrá configurar la personalización de la futura página web.
También habrá otra página que incluirá una sección de historial, en la que se 
guardarán todas las páginas generadas anteriormente por el usuario, así como 
borradores o procesos no finalizados, permitiendo su consulta o reutilización en el 
futuro
2. Validación y procesamiento del enlace de la API
Se leerá y verificará el enlace proporcionado por el usuario para comprobar su 
validez. Una vez validado, se procederá al análisis y procesamiento de los datos 
que devuelve la API.
3. Lectura de las preferencias de personalización
Una vez verificada la API se recogerán todas las opciones seleccionadas por el 
usuario para personalizar la página web:
Tema visual: Claro, oscuro o modo intercambiable.
Idioma: Posibilidad de elegir entre varios idiomas.
Estructura y diseño: Número de páginas a generar, estructura (horizontal, vertical, 
responsive...).
Tipografía: Tipo y tamaño de fuente.
Paleta de colores: Selección de colores preferidos.
Estilo general: Minimalista, clásico, moderno, con tarjetas, etc.
4. Generación de la página web personalizada
A partir de los datos procesados y la configuración seleccionada, se generarán los 
archivos necesarios para construir y desplegar la página personalizada

Contestacion del profesor: 

Hola Alejandro,
Perdona por la tardanza en contestar, ha sido una semana un poco complicada.
Gracias por el resumen. La base del proyecto me parece buena, Pero es verdad que para que el TFG destaque y tenga un componente claramente innovador, me gustaría que incorporáramos un módulo de IA que sugiera y genere el código de Django de la aplicación a partir de la API que indique el usuario y de sus preferencias.
Te dejo brevemente como lo veo:

    Convertir WebBuilder en un asistente que, además de generar la web personalizada, proponga y cree el backend en Django (modelos, vistas/URLs, templates y tests) de forma automatizada.
    El usuario introduce la URL de la API y las preferencias (tema, estructura, etc.).
    WebBuilder orquesta una comunicación con la IA por etapas (habría que construir bien los prompts):
        Primero genera una estructura y las apps necesarias
        Genera modelos/vistas/templates/tests
    Se valida la aplicación con pep8 y corriendo los tests
    Levanta el proyecto en un contenedor efímero
    Guardamos en WebBuilder todo, prompts, salidas y artefactos de cada build que se haga.

Yo creo que esto es realmente interesante, ya que con solo una API una descripción hecha con lenguaje natural tendríamos una aplicación funcional generada automáticamente.
Para usar la IA, puedes usar la API de openrouter que tiene muchos modelos (gratuitos aun que con un limit rate), la de together AI o si te ves con fuerzas puedes lanzarte tu propio modelo en local.
Dime que te parece, esto es la chispa que necesita el TFG, ya que ahora mismo todo gira entorno a la IA y es una buena manera de que tu también aprendas y te muevas en entornos que son el presente y serán el futuro.
Saludos,

Mini procesos
