Proceso

Cómo Funciona

Cuatro pasos para pasar de datos crudos a una web funcional

1

Introduce tu API

Pega la URL de cualquier API que devuelva JSON o XML. WebBuilder la descarga, la parsea y detecta automáticamente su estructura interna.

2

La IA genera el schema

Un LLM analiza los campos disponibles y decide cuáles mostrar, en qué orden y con qué etiquetas. Tú puedes ajustarlo campo a campo en el editor visual.

3

Diseño generado por IA

Otro LLM genera el HTML y CSS adaptado a tus datos reales. Puedes regenerar el diseño tantas veces como quieras con un simple hint de estilo.

4

Descarga el proyecto

Obtendrás un proyecto Django completo con models, views, templates y datos pre-cargados. Listo para ejecutar y desplegar en cualquier servidor.

Inteligencia Artificial

La IA que trabaja por ti

Dos modelos especializados colaboran para entender tus datos y construir la web perfecta para ellos

🧠

Planner — Entiende los datos

Analiza los campos del dataset y decide cuáles son relevantes para mostrar al usuario. Genera un schema dinámico libre: no hay campos fijos, el modelo elige qué mostrar según el contexto real de los datos.

Schema dinámico Anti-alucinación Validación en backend
🎨

Themer — Diseña la web

Recibe el schema validado y genera templates Django reales: HTML semántico, CSS adaptado al tipo de contenido e ítems de ejemplo. El diseño se puede regenerar con un hint de estilo en cualquier momento.

FAQ

Preguntas Frecuentes

Cualquier API pública que devuelva JSON o XML. El sistema detecta automáticamente el formato y la estructura. Puedes usar APIs REST, feeds XML, exports de datos, etc.
Usamos dos modelos LLM via OpenRouter. El primero (Planner) analiza los campos del dataset y genera un schema dinámico. El segundo (Themer) recibe ese schema y genera el HTML y CSS adaptado. Ambos tienen validación en backend para evitar alucinaciones.
Sí, puedes regenerar el diseño tantas veces como quieras desde el editor. Solo escribe un hint de estilo ("más oscuro", "estilo magazine", "minimalista") y la IA genera un nuevo diseño manteniendo tus datos y schema intactos.
Un proyecto Django completo con manage.py, settings.py, models.py con los campos de tu schema, views.py, urls.py y los templates generados por la IA. Los datos de la API vienen pre-cargados en la base de datos.
La integración con n8n está en el roadmap del proyecto. El objetivo es exponer una API REST de WebBuilder que n8n pueda llamar desde cualquier flujo de trabajo, permitiendo la generación automática de webs desde cualquier fuente de datos.
